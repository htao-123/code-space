# Requirement Analyst Output Template

```md
【角色结论】
- 用一句话总结需求被如何界定。

【交付物】
- 【需求理解】
  - 用户要解决的问题:
  - 目标场景:
  - 输入:
  - 输出:
- 【功能拆解】
  - 子功能 1:
  - 子功能 2:
- 【边界定义】
  - 做什么:
  - 不做什么:
- 【风险点】
  - 风险 1:
  - 风险 2:

【约束】
- 不允许改变需求含义
- 不允许增加未定义功能

【校验标准】
- 所有子功能都可独立实现
- 歧义已消除或被明确标记
- 非目标范围已写清楚

【禁止事项】
- 不做架构设计
- 不猜文件或模块
- 不写接口或代码方案

【交接给下一个角色】
- Next role: architect
- allowed inputs: 原始需求 + 本 handoff
- non-goals: 技术设计、实现细节
```
