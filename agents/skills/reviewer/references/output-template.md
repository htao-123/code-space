# Reviewer Output Template

```md
【角色结论】
- 用一句话总结实现是否通过审查，以及最大风险是什么。

【已核实输入】
- 当前角色：审核员
- 当前角色标识：reviewer
- 当前交接标识：
- 需求标识：
- 项目落点：
- 下一角色标识：tester
- 原始需求：
- solution handoff：
- implementer handoff：
- 当前任务文档：

【调研发现】
- 已检查改动范围：
- 已核实风险上下文：
- 是否需要外部调研：
- 外部调研来源：
- 外部调研结论：
- 内部证据清单：例如 `EVID-IN-001 -> path/to/file`
- 外部证据清单：例如 `EVID-EX-001 -> references/external/doc-snapshot.md | https://example.com/doc`
- 事实清单：使用 `FACT-001 -> 证据摘录：摘录内容` 格式，不夹带额外结论
- 证据映射：例如 `FACT-001 -> EVID-IN-001::keyword::摘录`
- 推断说明：
- 未验证项：

【交付物】
- 【问题列表】
  - 问题 1:
  - 严重性:
  - 证据:
  - 问题 2:
  - 严重性:
  - 证据:
- 【风险评估】
  - 残余风险 1:
  - 残余风险 2:

【约束】
- tester 必须覆盖主要风险和未决问题
- 不得把 review 通过当作功能正确的证明

【校验标准】
- 所有改动区域都已检查
- 发现项基于证据
- 即使无 bug 也写清残余风险

【禁止事项】
- 不静默改代码
- 不重写设计
- 不做空泛通过结论

【交接给下一个角色】
- 下一角色：测试员【负责验证正常路径、异常路径与边界情况】
- 可用输入：原始需求 + solution handoff + changed code + 本 handoff
- 非目标：直接改代码
- 完成条件：每个主要风险都有测试或阻塞说明
```
