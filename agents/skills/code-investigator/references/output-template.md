# Code Investigator Output Template

```md
【角色结论】
- 用一句话总结当前代码事实和主要证据结论。

【已核实输入】
- 原始需求：
- architect handoff：
- 当前任务文档：

【调研发现】
- 已检查文件：
- 已核实调用入口：
- 仍未知但需要保持未知的点：
- 是否需要外部调研：
- 外部调研来源：
- 外部调研结论：

【交付物】
- 【相关代码】
  - 文件 1:
  - 相关原因:
  - 文件 2:
  - 相关原因:
- 【调用链】
  - 入口 -> 中间层 -> 核心逻辑
- 【已有实现】
  - 类似实现 1:
  - 可复用点:
- 【状态流】
  - 数据/状态如何流转:
- 【潜在问题点】
  - 证据化问题 1:
  - 证据化问题 2:

【约束】
- 仅使用已列文件，除非发现新证据
- 不允许虚构不存在逻辑

【校验标准】
- 每个结论都有代码依据
- 调用链覆盖足够完整
- 未知项被明确标记为未知

【禁止事项】
- 不给解决方案
- 不排序方案优先级
- 不写代码

【交接给下一个角色】
- 下一角色：方案设计师【负责基于已验证事实提出最小可行解决方案】
- 可用输入：原始需求 + architect handoff + 本 handoff
- 允许文件/模块：仅限已验证范围
- 非目标：直接编码
- 完成条件：下一角色已拿到完整的事实证据与范围约束
```
